### Topic B - Building Redundant Switched Topologies
- Redundantcy avoids a single point of failure
- A redundant switch topology causes broadcast storm, multiple frame copies, and MAC address table instability problems
- A loop-avoidance mechanism is required  
  
- Избыточность позволяет избежать единой точки отказа
- Избыточная топология коммутатора вызывает широковещательный шторм, множественные копии кадров и проблемы нестабильности таблицы MAC-адресов
- Требуется механизм избежания петли   

spanning tree protocol  
протокол связующего дерева  
  
Root Bridge  
Root Port  
Designated Bridge  
Bridge ID = Bridge priority + MAC

### 5  
FHRP (First Hop Redundancy Protocol) — семейство протоколов, предназначенных для создания избыточности шлюза по умолчанию.  
HSRP (Hot Standby Router Protocol, Hot Standby Redundancy Protocol) — протокол маршрутизации семейства FHRP (англ. First-hop redundancy protocols), разработанный компанией Cisco и стандартизованный в RFC 2281.  
VRRP (Virtual Router Redundancy Protocol) — сетевой протокол, предназначенный для увеличения доступности маршрутизаторов, выполняющих роль шлюза по умолчанию.  
GLBP (Gateway Load Balancing Protocol) — проприетарный протокол Cisco, предназначенный для увеличения доступности маршрутизаторов выполняющих роль шлюза по умолчанию и балансировки нагрузки между этими маршрутизаторами.
